# MANUAL — How to Use DAF

Data Analysis Framework. Template for new data analysis projects. Reproducible, AI-friendly, opinionated layout.

---

## TL;DR

```bash
# 1. Clone or copy this template, then enter the project dir
cd your-project-name

# 2. Create env via uv (reads .python-version)
uv sync

# 3. Add the packages you need (template ships with empty deps)
uv add pandas pyarrow jupyterlab matplotlib python-dotenv

# 4. Start analysis
uv run jupyter lab notebooks/00_eda_template.ipynb
```

---

## Folder Structure

```
your-project/
├── .claude/
│   └── skills/             ← Claude Code skills (auto-triggered)
├── data/                   ← all datasets (gitignored content)
│   ├── raw/                  · original source data — never modify
│   ├── processed/            · cleaned/transformed outputs
│   └── external/             · reference / lookup / third-party
├── docs/                   ← documentation
│   └── data_dictionary.md    · auto-generated by `make snapshot`
├── notebooks/              ← Jupyter notebooks (numbered: 00_, 01_, …)
├── queries/                ← SQL by stage
│   ├── exploratory/          · ad-hoc investigation
│   ├── transformations/      · cleaning, reshaping, joining
│   └── final/                · production / report-feeding
├── reports/                ← deliverables
│   └── REPORT.md             · 14-section structured report
├── scripts/                ← reusable Python helpers
│   ├── paths.py              · path resolver (always import from here)
│   └── snapshot.py           · `make snapshot`
├── visuals/                ← exported charts, screenshots, ERDs
├── .env.example            ← copy to .env (gitignored), fill secrets
├── .gitignore
├── .python-version         ← pinned Python version
├── CLAUDE.md               ← context for AI assistants
├── LICENSE
├── MANUAL.md               ← this file
├── Makefile                ← `make help` lists targets
├── README.md               ← one-pager (stakeholder-facing)
├── SKILL.md                ← AI prompt reference for DA
└── pyproject.toml          ← uv-managed dependencies
```

Each top-level folder has its own `README.md` explaining what goes inside.

## Workflow

### 1. Set up environment

```bash
uv sync                                                  # creates .venv from .python-version
uv add pandas pyarrow jupyterlab matplotlib python-dotenv  # add what you need
```

### 2. Add data

Drop source files into `data/raw/`. Document the source in `README.md` ("Data Source" section).

```bash
make snapshot   # auto-generates docs/data_dictionary.md from raw/processed
```

### 3. Explore

Open `notebooks/00_eda_template.ipynb`. Always import paths via `scripts/paths.py`:

```python
from scripts.paths import raw, processed, visual

df = pd.read_csv(raw("source.csv"))
df_clean.to_parquet(processed("source_clean.parquet"))
fig.savefig(visual("dist_by_segment.png"), dpi=150)
```

**Never hardcode paths.**

### 4. Write queries

Place SQL in correct subfolder: `exploratory/`, `transformations/`, or `final/`. Naming: `001_explore_users.sql`, `002_transform_events.sql`.

### 5. Draft the report

- **Light projects**: fill `README.md` only — it's the one-pager + mini-report.
- **Heavy projects**: also fill `reports/REPORT.md` (14 sections, fixed order). Use prompts in `SKILL.md` to draft sections from raw findings.

### 6. Update README

`README.md` is always the entry point. Stakeholders read it first.

## AI Integration

Three layers:

1. **`CLAUDE.md`** — context Claude reads automatically. Conventions, paths, naming.
2. **`SKILL.md`** — prompt reference card for DAs. Copy-paste prompts for tasks not covered by skills.
3. **`.claude/skills/`** — Claude Code skills. Auto-trigger on relevant DA tasks (`profile-data`, `draft-insight`, `write-sql`).

When Claude works in this repo, it follows `CLAUDE.md`. Skills in `.claude/skills/` activate on matching prompts. `SKILL.md` is for the DA, not Claude.

## Reproducibility

- **Python version** pinned in `.python-version`
- **Deps** locked in `uv.lock` (commit it after `uv sync`)
- **Paths** resolved via `scripts/paths.py`, anchored on `pyproject.toml` — works from any cwd
- **Secrets** in `.env` (gitignored), template in `.env.example`
- **Data files** gitignored — share via separate channel + document source in `README.md`

| Target | What it does |
|--------|--------------|
| `make install` | `uv sync` — install/update deps |
| `make snapshot` | Generate `docs/data_dictionary.md` from data files |
| `make help` | List all targets |

## Conventions

- **Notebooks**: `NN_purpose.ipynb` (numbered, snake_case)
- **Queries**: `NNN_action_object.sql`
- **Visuals**: `purpose_dimension.png` (e.g. `churn_by_region.png`)
- **Processed data**: `entity_state.parquet` (e.g. `customers_clean.parquet`)
- **Commits**: imperative, scoped (`feat:`, `fix:`, `docs:`, `data:`)

---

## Customizing

- Drop folders you don't need (e.g. no SQL → delete `queries/`)
- Add packages with `uv add <pkg>` — keep dep list minimal
- Extend `.claude/skills/` with project-specific helpers
- Update `CLAUDE.md` when conventions diverge from defaults
