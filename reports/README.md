# Reports

Final deliverables and the canonical structured report.

## What goes here

| File | Purpose |
|------|---------|
| `REPORT.md` | Full structured report. 14 sections, fixed order. Fill section by section. |
| `*.pdf`, `*.docx`, `*.pptx` | Polished deliverables for stakeholders. |
| `*_summary.md` | Optional shorter recaps for specific audiences. |

## Conventions

- `REPORT.md` is the **single source of truth** for the analysis narrative. Render to PDF/Word from this when sharing.
- Polished deliverables (PDF, slides) are exports — don't author insights here.
- Naming: `[project-slug]_[deliverable]_[period].ext` — e.g. `churn-q3_summary_2024Q3.pdf`.

1. Project Overview
2. Objectives
3. Scope & Tools

7. ERD
